# 使用指南

## App 是什么

灵动键 FlexKey 用规则把系统事件和自动化动作连接起来。它适合这些场景：

- 把音量键、蓝牙按键、键盘按键映射成返回、主页、媒体控制或打开指定页面。
- 通过悬浮球手势快速执行动作。
- 监听通知或界面事件，匹配关键词后自动执行动作。
- 用 JS 脚本编写更精细的触发条件和动作。
- 通过 Shizuku 增强按键/触控模拟、截图、强制停止应用等能力。

## 初始配置

进入“设置 > 服务与权限”，按实际需要开启：

| 设置项 | 什么时候需要 |
| --- | --- |
| 无障碍服务 | 使用按键条件、组合按键条件、无障碍事件条件、全局控制、无障碍节点动作、部分输入模拟 |
| 通知服务 | 使用通知事件条件、通知 JS 条件、通知控件点击动作 |
| 悬浮球 | 使用悬浮球条件 |
| Shizuku | 使用增强按键/触控模拟、Shizuku 截图、强制停止等高权限动作 |
| 自启动/关联启动/后台弹窗 | 让后台服务更稳定，厂商系统需要手动确认 |
| 修改系统设置 | 使用自动旋转、亮度、屏幕方向等动作 |
| 忽略电池优化 | 降低后台服务被系统杀掉的概率 |

权限申请前，App 会显示用途说明并等待用户明确同意，然后跳转到系统设置页。

## 规则模型

规则由“条件 + 生效范围 + 动作”组成。

### 条件

创建规则时条件有且只能选择一个：

- 按键：监听实体按键、蓝牙按键、键盘按键等，支持单击、长按、双击、三击。
- 组合按键：多个按键组合触发。
- 虚拟 HOME：部分设备上的虚拟 HOME 事件，支持双击、三击。
- 悬浮球：支持单击、双击、长按、向左/上/右/下滑动。
- 手势识别：在手势识别界面画出预设手势后触发。
- 事件引擎：监听无障碍事件或通知事件，通过文本、控件 ID、事件类型等简单字段匹配。
- JS 脚本：把按键、悬浮球、通知、无障碍事件输入给脚本，脚本返回 `true` 时触发动作。

### 生效范围

生效范围有两种：

- 全局：任何前台应用都可触发。
- 特定应用内：只在当前前台包名匹配所选应用时触发。

特定应用的判断使用当前前台应用包名。规则保存时会记录应用包名和显示名称，执行时按包名匹配。

### 动作

动作可以从动作库选择，也可以在创建规则时直接创建新动作。创建新动作支持：

- 打开应用。
- 打开快捷方式。
- 模拟通知控件点击。
- 模拟按键。
- 自定义动作。
- JS 动作。

新动作创建完成后会自动填入当前规则，不需要再选择一次。

## 创建规则流程

1. 打开“规则”页，点击创建。
2. 选择一种条件。
3. 设置条件参数，例如按键类型、事件来源、匹配文本、JS 脚本等。
4. 选择生效范围：全局或特定应用。
5. 选择动作，或点击“创建新动作”。
6. 保存规则。

## 管理规则

规则列表支持：

- 开关：启用或停用规则。
- 点击：打开规则菜单。
- 长按：拖动调整规则顺序。
- 更改动作：重新进入创建规则页，只修改动作部分。
- 删除：删除前会要求确认。

## 条件说明

### 按键条件

按键条件依赖无障碍服务。创建时按下目标按键，App 会记录：

- keyCode。
- keyName。
- 输入设备 ID、名称、描述符。
- 是否忽略设备信息。
- 触发类型：单击、长按、双击、三击。

忽略设备信息后，只要 keyCode 相同就能触发；否则同一按键来自不同设备时会被区分。

### 悬浮球条件

悬浮球依赖悬浮窗权限。事件类型：

- `SingleTap`：单击。
- `DoubleTap`：双击。
- `LongPress`：长按。
- `FlingLeft`：向左滑动。
- `FlingUp`：向上滑动。
- `FlingRight`：向右滑动。
- `FlingDown`：向下滑动。

### 事件引擎

事件引擎用于不写代码的简单场景。

无障碍事件可匹配：

- 事件类型：所有事件、窗口变化、控件点击、文本变化、焦点变化。
- 匹配文本：标题、内容、控件文本、类名等聚合文本，大小写不敏感。
- 控件 ID：例如 `com.example:id/button`。

通知事件可匹配：

- 通知来源应用。
- 标题、正文、展开正文、副标题等聚合文本。

事件引擎规则对同一规则有 1 秒触发间隔，避免通知或界面频繁刷新时重复触发。

### JS 脚本条件

JS 条件适合复杂判断。脚本会收到 `event` 和 `input` 两个对象，它们是同一个事件对象。

只有脚本明确 `return true;` 才会触发动作。返回其他 truthy 值不会触发。

示例：

```js
if (event.source !== 'notification') {
    return false;
}

return event.packageName === 'com.example.chat'
    && event.text.indexOf('验证码') >= 0;
```

详见 [JS 脚本与条件 API](js-api.md)。

## 动作反馈

在“设置 > 其他设置”中可以配置：

- 执行成功后震动。
- 执行成功后显示动作名称。

动作执行失败时，如果开启调试信息，App 会显示错误消息。

## 稳定性建议

- 长期使用按键监听、悬浮球、通知监听时，建议开启自启动、关联启动、后台弹窗和忽略电池优化。
- 厂商权限页面差异很大，App 会尽量跳转到对应页面；如果无法判断状态，需要在系统页面手动确认。
- JS 条件和服务内部异常会被捕获并记录，不应导致无障碍服务或通知服务整体崩溃。
