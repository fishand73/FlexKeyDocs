# 隐私政策

生效日期：2026 年 5 月 12 日

本隐私政策适用于灵动键 FlexKey Android 应用和本文档站点。灵动键是一款本地自动化工具，核心功能依赖用户主动创建的规则、条件、动作和脚本。

## 我们默认不会上传的内容

默认情况下，灵动键不会把以下内容上传到开发者服务器：

- 您创建的规则、条件、动作、脚本和手势配置。
- 您选择的应用范围、动作库安装状态和本地设置。
- 无障碍事件、通知内容、按键事件、截图内容或前台应用信息。

这些数据主要保存在应用私有数据目录中，用于在本机执行自动化规则。

## 本地处理的数据

为实现功能，灵动键会在设备本地处理以下数据：

| 数据类型 | 用途 |
| --- | --- |
| 规则、动作和脚本 | 保存用户配置，并在条件满足时执行动作 |
| 应用包名和应用名称 | 设置规则生效范围、打开应用、检查内置动作依赖 |
| 按键事件和输入设备信息 | 识别实体按键、蓝牙按键和组合按键条件 |
| 无障碍事件和界面文本 | 匹配无障碍条件，执行返回、主页、滚动、点击等自动化动作 |
| 通知标题、正文和通知按钮 | 匹配通知条件，执行通知控件动作 |
| 截图或媒体投影结果 | 仅在用户触发截图相关动作时使用 |
| 远程动作库 URL | 保存用户主动添加的动作库地址 |

## 敏感权限的用途

灵动键只在用户启用相关功能时请求或引导开启权限：

- 无障碍服务：用于按键监听、无障碍事件条件、全局返回/主页/最近任务、滚动、复制粘贴和部分触控模拟。
- 通知使用权：用于通知条件、通知文本匹配和通知按钮点击。
- 悬浮窗权限：用于显示悬浮球并接收悬浮球手势。
- Shizuku 授权：用于增强按键/触控模拟、截图、强制停止应用等高权限动作。
- 查询已安装应用：用于选择规则生效应用、打开应用、检查内置动作依赖状态。
- 前台服务和通知权限：用于在用户开启相关开关后维持必要的后台能力并显示服务状态。
- 忽略电池优化、自启动和厂商后台权限：用于提高长期监听和后台服务稳定性。
- 修改系统设置、勿扰权限、相机/手电筒、拨号等权限：用于用户选择的对应系统动作。

权限申请前，应用会展示用途说明并等待用户确认。您可以随时在系统设置中撤销这些权限。

## 网络访问

灵动键的网络访问主要发生在以下场景：

- 加载您主动添加的远程动作库 URL。
- 打开帮助文档、反馈页面或应用市场页面。
- 某些分发渠道或构建变体如果启用了崩溃反馈，会按该渠道的说明处理诊断信息；本文档对应的默认行为是不主动上传本地规则、脚本或事件内容。

当您添加远程动作库时，请确认来源可信。远程服务器可能接收常规网络请求信息，例如 IP 地址、请求时间、User-Agent 和请求路径。

## 第三方服务和外部链接

文档、反馈、应用市场、远程动作库和您通过动作打开的第三方应用由各自服务提供方控制。灵动键无法控制这些第三方如何处理数据。离开应用或本文档站点后，请查看对应服务的隐私政策。

## 数据共享

灵动键不会出售个人数据。应用执行用户配置的动作时，可能会把必要信息交给 Android 系统或目标应用，例如打开指定应用页面、发送 Intent、拨打电话、点击通知按钮或访问用户添加的远程动作库。

## 数据保留和删除

本地规则、动作、脚本和设置会保留到您删除它们、清除应用数据或卸载应用为止。撤销无障碍、通知、悬浮窗、Shizuku 等权限后，相关功能会停止使用对应权限。

## 儿童隐私

灵动键不是面向儿童的应用。未成年人使用本应用时，应获得监护人同意并遵守当地法律要求。

## 政策更新

如果应用功能、数据处理方式或权限用途发生重要变化，我们会更新本文档，并在页面顶部标明新的生效日期。
